# Sugar News Task Instructions

本目录是独立的 Sugar News 工作任务，不属于任何其他项目。执行 Sugar News 时，只能在本目录内新增或更新 Sugar News 相关文件，除非用户明确要求同步到其他项目。

## 任务目标

每天联网检索并整理上一自然日发布或发生的全球糖业新闻，最终按照本目录固定 Excel 模板生成简洁日报。

- 目标新闻日期 = 系统当前日期的上一自然日。
- 例如 2026-07-20 运行，则整理 2026-07-19 发布或发生的新闻。
- 新闻发布日期按新闻所涉及国家或地区的当地时间判断。
- 不得仅依靠历史知识、旧数据、本地缓存或模型记忆生成日报。

## 固定执行路径

每次运行必须依次完成以下步骤：

1. 联网检索并记录检索日志。
2. 核对发布日期、事件日期、来源和正文相关性。
3. 将通过核验的新闻保存为结构化清单。
4. 由脚本根据清单写入 Excel。
5. 保存后重新读取 Excel，逐条核对清单与表格内容。

标准文件路径：

```text
logs/YYYY/MM/search_log_YYYY-MM-DD.json
data/verified_news/YYYY/MM/sugar_news_YYYY-MM-DD.json
reports/YYYY/MM/Sugar News YYYY-MM-DD.xlsx
logs/YYYY/MM/write_log_YYYY-MM-DD.json
```

已核验新闻必须全部进入 Excel。如果核验清单中存在某国新闻，但保存后的 Excel 没有该国对应行，生成失败并必须自动修复。

- 目标日期；

- 请求状态；

如果某个来源无法访问，应继续使用其他权威来源，不得因为单个网站失败就判断整个国家没有新闻。不得因为一次英文检索无结果就省略整个国家。

## 新闻范围

重点检索：

1. 甘蔗种植面积、单产、产量及天气影响；
2. 甘蔗压榨量、糖产量、出糖率和制糖比例；
3. 乙醇产量、价格、掺混政策和甘蔗制醇比例；
4. 糖出口、进口、配额、关税和贸易政策；
5. 国内糖价、国际糖价和主要糖厂报价；
6. 糖厂开榨、停榨、扩建、检修及经营情况；
7. 政府政策、行业协会声明和官方数据；
8. 可能影响全球糖业供需或 ICE 原糖价格的其他事件。

中国新闻重点覆盖甘蔗及甜菜种植面积/单产/产量，广西、云南、广东、海南、新疆、内蒙古等主要产区天气，糖厂开榨/收榨/压榨量/产糖量/出糖率/产销率，全国及产区库存，食糖进口量/来源/价格/海关数据，配额内外进口政策、关税、配额发放、进口管理政策，白糖进口成本、加工利润、内外价差，郑州商品交易所白糖期货行情、仓单和持仓变化，国内现货价格、基差、区域报价，糖浆/预混粉及替代品进口，企业经营、国家收储/抛储/政策性库存和协会/政府/企业产销数据。

中国新闻不得收录普通食糖健康消费文章、甜品/饮料/食品营销新闻、与糖业基本面无关的价格转载、只有行情涨跌且无基本面原因的盘面复述、与甘蔗主产区无关的普通天气、旧闻和重复报道。

有效新闻必须根据正文内容判断，不得只用英文标题关键词筛选。葡萄牙语 `açúcar`、`cana`、`etanol`，印地语 `चीनी`、`गन्ना`、`इथेनॉल`，泰语 `น้ำตาล`、`อ้อย`、`เอทานอล` 都应识别为糖业相关表达。

天气新闻只有在明确涉及主要甘蔗产区，并可能影响甘蔗生长、收割、压榨或运输时才能收录。

### 印度甘蔗产区降雨监测

每日印度新闻检索必须新增“印度甘蔗产区降雨监测”，并作为固定流程执行。每天检索：印度昨日实际降雨情况；未来 7-15 天降雨预报；新闻发布日期可以是昨日或今日，但天气事件必须与昨日实况或最新预报有关；发布日期和天气发生时间按印度当地时间核对。

重点监测印度主要甘蔗产区：

- 核心监测地区：北方邦（Uttar Pradesh）、马哈拉施特拉邦（Maharashtra）、卡纳塔克邦（Karnataka）。
- 其他主要产区：泰米尔纳德邦（Tamil Nadu）、古吉拉特邦（Gujarat）、比哈尔邦（Bihar）、旁遮普邦（Punjab）、哈里亚纳邦（Haryana）、北阿坎德邦（Uttarakhand）。

若报道细化到地区或城市，必须判断其是否位于甘蔗主产带；不得只按邦名粗略判断。

印度降雨检索不得只使用通用新闻搜索，必须同时检查：印度气象局（IMD）的每日降雨报告、季风报告、天气预警、扩展期预报和农业气象公告；印度各邦气象部门、农业部门及甘蔗管理部门；印度英文和印地语主流媒体；糖业、农业、季风及大宗商品专业媒体；必要时查询天气模型或可靠气象平台，用于交叉验证未来降雨趋势。

印度降雨新闻收录规则：

- 如果昨日实际降雨或未来降雨预报涉及主要甘蔗产区，并可能影响甘蔗生长、单产、收割、压榨进度或糖产量，必须作为印度糖业相关新闻补充到报告中。
- 不得因为已有其他印度糖业新闻而省略天气新闻，也不得因同一国家篇幅较长而删除具有独立影响的信息。
- 同一轮天气过程被多家媒体重复报道时，应合并为一条；如果不同产区的降雨方向或影响明显不同，可以分别列出。
- 降雨地区不属于主要甘蔗产区，或报道未说明其对甘蔗生产的影响时，判断为影响有限或不收录。

印度降雨影响判断：

- 甘蔗生长阶段出现适量降雨、降雨预报增多、强降雨或暴雨预报但未证实受灾、土壤墒情改善：判断为“偏空糖价：降雨有利于甘蔗生长和单产提升，可能增加未来甘蔗及食糖供应。”
- 干旱、降雨不足、季风偏弱或未来降雨预报减少：判断为“偏多糖价：水分不足可能降低甘蔗单产和糖料供应。”
- 已确认发生洪涝、农田被淹、甘蔗倒伏、道路中断、作物受损或预期减产：判断为“偏多糖价”，并说明实际损失依据。
- 收割或压榨阶段降雨导致砍蔗、运输或糖厂开榨受阻：判断为“偏多糖价”或“短期偏多糖价”。
- 不得仅根据“强降雨”“暴雨”等词语机械判断，必须结合甘蔗所处生长阶段、产区位置以及是否已经造成实际损失。

印度降雨新闻输出时，每条新闻用两句话：第一句概括昨日降雨实况或未来降雨预报，包括时间、地区、降雨趋势及关键数据；第二句给出对糖价的影响方向和判断逻辑。B 列仍必须保留来源链接，C 列仍必须以四类影响判断之一开头。

### 泰国甘蔗产区天气判断

泰国主要甘蔗种植区名单必须用于识别泰国天气新闻：

- 东北部：乌隆他尼（Udon Thani）、孔敬（Khon Kaen）、呵叻/那空叻差是玛（Nakhon Ratchasima）、猜也蓬（Chaiyaphum）、加拉信（Kalasin）、黎府（Loei）。东北部是泰国最核心甘蔗产区。
- 北部：那空沙旺（Nakhon Sawan）、甘烹碧（Kamphaeng Phet）、素可泰（Sukhothai）、彭世洛（Phitsanulok）。
- 中部及西部：北碧（Kanchanaburi）、华富里（Lopburi）、素攀武里（Suphanburi）、猜纳（Chai Nat）。
- 东部：沙缴（Sa Kaeo）、春武里（Chonburi）。

每日泰国新闻检索完成后，必须单独执行“泰国主要甘蔗产区降雨检查”，不能只依赖普通新闻搜索。固定检查最近一个完整自然日降雨实况、当日天气、未来 7 天降雨预报、降雨是否增加、是否存在强降雨或暴雨预警，以及是否存在连续少雨、高温或干旱风险。

泰国天气优先使用泰国气象局官方日预报：https://tmd.go.th/en/forecast/daily。如果官方只给出区域性信息，可以用公开天气预报补充具体甘蔗产区，但必须保留来源链接，不得编造降雨量或降雨概率。检索不能只搜索 `Thailand weather`，必须把上述主要府名与 `rain`、`rainfall`、`thunderstorm`、`heavy rain`、`monsoon`、`drought` 等天气词组合检查。

只要主要甘蔗产区存在有效降雨预报，泰国栏目必须增加一条天气新闻；即使当天没有其他泰国糖业新闻，也不得因缺少媒体报道而遗漏天气信息。同一天相同天气过程合并为一条，摘要用 2-3 句说明涉及产区、预报时间、降雨概率或雨量（来源提供时才写）、降雨类型和对甘蔗生长及糖价的影响。

每次整理泰国天气新闻时，必须依次确认：降雨发生或预报的具体府名；是否属于上述主要甘蔗种植区；当前处于甘蔗生长阶段还是收割压榨期；降雨是否实际出现、预报增加、提示强降雨，或多个主要产区的降雨范围/强度扩大；是否已经明确造成严重洪涝、甘蔗倒伏、农田被淹、作物受损或预计减产；最终应判断为偏多糖价、偏空糖价、中性或影响有限。

泰国天气影响判断规则：

- 甘蔗生长阶段，主要产区出现强降雨、大雨、雷阵雨、阵雨、预报大雨、预报强降雨、预报雷阵雨、降雨范围扩大、降雨概率提高、未来降雨量增加、连续降雨，或降雨缓解干旱、改善土壤墒情：统一判断为“利空：甘蔗生长阶段的降雨有利于补充土壤水分、改善墒情并促进甘蔗生长和单产形成，从而增加未来甘蔗及食糖供应预期，因此利空糖价。”
- 生长季降雨减少或持续干旱：判断为“偏多糖价：降雨不足可能抑制甘蔗生长并降低单产，增加未来糖产量下降风险。”
- 即使降雨具有局地性、持续时间较短、未覆盖全部甘蔗产区、尚未达到全国性强降雨，或仍为预报而非实况，只要覆盖上述主要甘蔗产区且处于甘蔗生长阶段，均判断为利空糖价，不得弱化判断。
- 只有新闻明确指出降雨已经造成严重洪涝、甘蔗倒伏、农田被淹、作物受损或预计减产时，才根据实际损失重新判断；不得仅凭“山洪风险”“强降雨风险”等预报性文字改判为偏多。
- 收割及压榨期降雨增加：不得套用“降雨有利生长”，原则上判断为“偏多糖价：收割及压榨期降雨可能影响甘蔗收割、运输和糖厂入榨进度，造成短期供应延迟。”
- 降雨只出现在非甘蔗主产区：判断为“影响有限：降雨区域与泰国主要甘蔗种植区重合度较低，对全国甘蔗及糖产量影响有限。”

参考写法：

新闻：泰国气象部门预计，未来一段时间乌隆他尼、孔敬、呵叻和猜也蓬等东北部主要甘蔗产区降雨将增加，有利于改善当地土壤墒情。

影响：利空：甘蔗生长阶段的降雨有利于补充土壤水分、改善墒情并促进甘蔗生长和单产形成，从而增加未来甘蔗及食糖供应预期，因此利空糖价。

如果现有或后续报告中出现“泰国东北部和中部主要甘蔗产区降雨增加、未来降雨量增加、可能出现强降雨、雷阵雨、阵雨、大雨或降雨范围/强度扩大”，且处于甘蔗生长阶段、未明确发生作物损失，应统一判断为“利空：甘蔗生长阶段的降雨有利于补充土壤水分、改善墒情并促进甘蔗生长和单产形成，从而增加未来甘蔗及食糖供应预期，因此利空糖价。”不得再添加弱化判断。

## 新闻数量和国家顺序

新闻数量完全按照目标日期实际发生的重要糖业新闻确定，不规定各国家的新闻数量，不设最低数量和最高数量。

- 同一国家当天有多少条重要且不重复的糖业新闻，就整理多少条。
- 每条新闻单独占一行。
- 某个国家当天没有重要糖业新增信息时，不生成该国家的数据行。
- 不得填写“暂无新闻”“暂无最新数据”等占位内容。
- 不得用旧闻、重复报道、低相关性内容、普通食品或健康消费新闻补足数量。

国家整体排列顺序必须严格保持：

1. 中国；
2. 巴西；
3. 印度；
4. 泰国；
5. 其他国家。

中国新闻数量不设最低数量和最高数量。有多少条符合规则的重要新闻就整理多少条；若目标日期没有重要新增信息，直接省略中国数据行，不写占位内容。

“其他国家”同样不限制新闻数量，按照新闻重要程度排列，并在 A 列直接填写具体国家或地区名称，例如“美国”“欧盟”“墨西哥”，不要统一写成“其他”。

## 新闻摘要、国家归属和非糖业排除

新闻摘要必须使用中文糖业研究报告表达，每条提炼为 2-3 句，说明核心事件、关键数据或政策变化，以及对食糖供需、甘蔗生产、乙醇分流、进出口、库存或糖价的影响。不得逐句翻译英文标题；不得添加原文没有的数据；不得使用宣传性、背景铺陈或重复表述。页面已有报告日期时，新闻正文不要机械重复“今日”“本日发布”“X月X日消息”或普通发布日期；只有政策起止、榨季、配额、统计截止、降雨预报覆盖期、开榨停榨等本身影响判断的日期才保留。

国家归属必须以新闻核心事件发生地和主要影响对象为准，不得按来源网站、转载媒体所在地、文章语言、企业总部所在地或标题中出现次数最多的国家机械分类。分类前必须核对核心国家、事件发生地、政策发布机构所属国家、企业或糖厂运营所在地和主要受影响市场。某国进口另一国食糖时，进口政策、采购量和国内供应归进口国；出口销售、出口配额和生产安排归出口国；涉及多国但无明确主体时归入“其他国家”下的实际国家或“全球”，不得在两个栏目重复发布同一条新闻。

Sugar News 只整理食糖产业相关内容。必须排除人体血糖、糖尿病、医学健康、营养保健、胰岛素、降糖药、血糖仪、连续血糖监测、控糖饮食、代糖健康研究等新闻。英文检索和自动过滤必须排除 `blood sugar`、`glucose`、`diabetes`、`insulin`、`glycemic`、`hyperglycemia`、`hypoglycemia`、`glucose monitoring`、`diabetes treatment` 等语义。游戏、小说、影视、音乐、餐饮营销、菜谱甜点、文学作品或普通消费内容不得仅因标题包含 `sugar`、`palm sugar`、`burnt sugar` 等词进入报告。不得仅因标题包含 `sugar` 就判断为糖业新闻。

乙醇新闻只有在与甘蔗、糖蜜、糖浆、糖醇比、糖业自给或食糖供应存在直接关系时才能收录。一般能源、汽油、玉米乙醇、汽车燃料新闻不得仅因出现“乙醇”“E10”“E20”自动收录；印度乙醇政策按本文件既有“糖业间接影响”规则处理。

结构化数据中每个其他国家必须使用独立对象或独立数组保存，不得使用单个 `其他` 键保存一条新闻，也不得让后一个国家覆盖前一个国家。同一其他国家有多条新闻时，A 列可以重复填写该国家名称，每条新闻单独占一行。

结构化数据中中国必须使用独立对象：

```json
{
  "country": "中国",
  "items": []
}
```

不得将中国保存到“其他”字段或 `country_group=其他国家`。

## Excel 模板

固定模板为：

```text
templates/新闻格式.xlsx
```

模板只作为三列结构和样式参考，不代表新闻数量要求。执行任务时必须复制模板后填写，不得直接覆盖或修改原始模板。

固定列：

- A 列：国家
- B 列：新闻
- C 列：影响

不得新增工作表或列，不得合并单元格，不得改变 A:C 三列结构。

## 写作规则

B 列“新闻”：

- 每条新闻使用一至两句话；
- 简洁说明事件、关键数据、政策或地区；
- 如有同比或环比，保留准确变化；
- 新闻来源必须放在正文末尾，格式为：`来源：机构或媒体名称（原始链接）`。

C 列“影响”：

- 每条影响控制在一句话；
- 必须以 `偏多糖价：`、`偏空糖价：`、`中性：` 或 `影响有限：` 之一开头；
- 事实与影响判断必须分开，不得把推测写成已经发生的事实。

判断中国新闻影响时，应区分对中国国内白糖现货价格、郑糖期货、中国进口需求和 ICE 原糖的影响。不得简单把所有中国利多消息都写成利多 ICE 原糖；可以使用类似“偏多糖价：国内减产预期支撑郑糖，但进口政策没有变化，对 ICE 影响有限”的表达。

当甘蔗收购价、最低支持价、FRP 或 SAP 提高时，如果主要作用是提高农户种植收益、刺激甘蔗面积扩张和增加未来糖产量，原则上判断为偏空糖价；如果同时存在明显的糖厂成本上升、停产或减产风险，则分别说明短期成本影响与中长期供应影响，不得直接判断为中性。

## 数据和单位

甘蔗、糖及进出口数量优先统一换算为“万吨”。例如 `273.90 LMT` 应写成 `2739万吨`，不得保留 `273.90 LMT` 或 `lmt`。

乙醇使用“万升”或“亿升”，同一份日报内保持统一。价格、比例、同比变化和日期必须保留准确数值。所有内容使用中文，语言专业、客观、简洁。

## 印度糖价与库存指标

印度新闻检索必须长期纳入三类独立指标：印度国内糖价、北方邦糖厂出厂价、印度食糖结转库存及变化。这些指标用于印度新闻判断、Excel新闻行（当日确有新价格或库存消息时）和 Sugar News 数据看板的“印度糖价与库存”卡片。

印度国内糖价每日优先检索 S-grade sugar、M-grade sugar、马哈拉施特拉邦、北方邦、卡纳塔克邦等主要市场或主产区代表价格，必须区分现货批发价、零售价、糖厂出厂价、政府最低销售价格和甘蔗收购价。北方邦糖厂出厂价重点检索 Muzaffarnagar、Meerut、Shamli、Saharanpur、Hapur、Bareilly 等报价中心，不得把北方邦甘蔗收购价当作食糖出厂价。

印度结转库存只在出现新的官方数据、协会预测、机构预测或可信媒体更新时作为新闻收录；没有新数据时，看板保留最近一期有效数据及其发布日期，不得每天重复生成相同库存新闻。必须区分期初库存、期末/结转库存、糖厂库存、政府库存、可出口余量和未售库存。

指标数据质量要求：
- 至少使用两个可靠来源交叉验证关键价格或库存数据，优先采用官方、行业协会、一手市场数据和可信糖业媒体。
- 必须保留价格日期、品种/等级、报价地区、当前价格、上一可比价格、涨跌金额、涨跌幅、原始单位、来源链接、发布日期和抓取时间。
- 不能获取可靠数据时，指标卡显示“数据待更新”，不得编造价格、库存或涨跌幅；没有新库存数据时不得生成重复库存新闻。
- 数据存在冲突时展示差异并说明采用口径；不同来源的榨季定义或库存口径不一致时不得直接比较。

单位换算必须由程序完成：
- `1 quintal = 100公斤`，`₹/kg = ₹/quintal ÷ 100`。
- `1 lakh tonne = 10万吨`，`1 million tonnes = 100万吨`。

影响判断规则：
- 印度国内糖价或北方邦出厂价上涨，原则上判断为“偏多糖价”，因为国内现货需求增强、供应趋紧或糖厂挺价可能支撑糖价。
- 印度国内糖价或北方邦出厂价下跌，原则上判断为“偏空糖价”，因为可能反映供应宽松、需求疲弱或糖厂销售压力增加。
- 印度结转库存上调或同比增加，原则上判断为“偏空糖价”；结转库存下调或同比下降，原则上判断为“偏多糖价”。
- 若变化幅度很小或已被市场充分预期，可以判断为“中性”或“影响有限”。影响方向必须结合产量、消费、出口政策、乙醇分流和政府调控综合判断。
